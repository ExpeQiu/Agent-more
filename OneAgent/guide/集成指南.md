# 集成指南

OneAgent 提供三种接入方式：**HTTP 服务**、**Node SDK 嵌入**、**CLI**。统一入口为 `OneAgentExecutor`，支持 standalone / kernel 双轨执行。

> 执行层级说明见 [双轨执行架构](./双轨执行架构.md) · Profile 配置见 [AgentProfile规范](./AgentProfile规范.md)

---

## 选型速查

| 宿主场景 | 推荐方式 | tier |
|---------|---------|------|
| 任意语言业务系统 | HTTP Sidecar | `standalone` 或 `auto` |
| Node/TS 同进程嵌入 | SDK | `standalone` + 自定义 HostBridge |
| 多步工具 / 联邦 / 审批 | HTTP 或 SDK | `kernel` |
| 本地调试 | CLI | `--tier standalone` |

---

## HTTP 服务部署

```bash
cd OneAgent
npm install
npm run build
./scripts/start.sh          # 默认 :8790
curl http://127.0.0.1:8790/healthz
```

环境变量：

| 变量 | 说明 |
|------|------|
| `OPENAI_API_KEY` | 真实模型推理 |
| `ONEAGENT_MOCK_MODE=true` | Mock 响应（无需 Key） |
| `ONEAGENT_TOKEN` | Bearer 鉴权 |
| `ONEAGENT_AUTO_APPROVE=true` | kernel tier 开发环境自动审批 |
| `COMPO_PLUGINS_CONFIG` | 联邦插件配置路径 |

### API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/healthz` | GET | 健康检查 |
| `/v1/agents` | GET | 列出 Agent Profile |
| `/v1/agents/:id` | GET | Profile 详情（含 execution） |
| `/v1/agents/:id/run` | POST | 同步执行，响应含 `executionTier` |
| `/v1/agents/:id/stream` | POST | SSE 流式（仅 kernel tier） |
| `/v1/chat` | POST | Copilot 多轮，自动 sessionId |

### 简单场景（standalone）

```bash
curl -X POST http://127.0.0.1:8790/v1/agents/copilot/run \
  -H 'Content-Type: application/json' \
  -d '{
    "tier": "standalone",
    "task": {
      "goal": "用三句话解释 REST 和 GraphQL 的区别",
      "actor": { "userId": "u1" }
    }
  }'
```

### 复杂场景（kernel）

```bash
curl -X POST http://127.0.0.1:8790/v1/agents/planner/run \
  -H 'Content-Type: application/json' \
  -d '{
    "tier": "kernel",
    "task": {
      "goal": "拆解 v2.1 发布流程为可执行步骤，并检索相关知识",
      "actor": { "userId": "orchestrator", "roles": ["operator"] },
      "contextRefs": [{ "kind": "document", "ref": "release:v2.1" }]
    }
  }'
```

### 自动路由（auto）

省略 `tier` 或传 `"tier": "auto"`，按 Profile `execution.defaultTier` 与 escalate 规则决定：

```bash
curl -X POST http://127.0.0.1:8790/v1/agents/reviewer/run \
  -H 'Content-Type: application/json' \
  -d '{
    "task": {
      "goal": "审阅 README",
      "actor": { "userId": "u1" },
      "contextRefs": [{ "kind": "document", "ref": "readme:v1" }]
    }
  }'
```

`reviewer` Profile 配置了 `escalateToKernelWhen.hasContextRefs: true`，有 contextRefs 时自动走 kernel。

### 联邦 Header

建议透传 compo-standards Header：

```text
Authorization: Bearer <token>
X-Tenant-Id: <tenant>
X-Platform-Id: <platform>
X-Correlation-Id: <session-or-task>
```

## SDK 嵌入宿主

适合 Node/TS 业务系统同进程集成，**可注入 HostBridge** 解析真实业务上下文。

```ts
import { createOneAgentRuntime } from "@compo/oneagent";

const runtime = await createOneAgentRuntime({
  hostBridge: {
    async resolveContext(refs, task) {
      return Promise.all(
        refs.map(async (ref) => ({
          kind: ref.kind,
          title: ref.title ?? ref.ref,
          content: await mySystem.loadDocument(ref.ref),
          priority: 10,
        })),
      );
    },
    async requestApproval(request) {
      const approved = await mySystem.requestUserApproval(request);
      return { approved, approverId: "user-ui" };
    },
    async audit(record) {
      await mySystem.logAudit(record);
    },
  },
});

const task = {
  taskId: `task_${Date.now()}`,
  actor: { userId: "biz-user" },
  goal: "审阅合同第三节",
  contextRefs: [{ kind: "document", ref: "contract:v3" }],
  metadata: { agentId: "reviewer" },
};

// 简单 — OneAgent 独立处理
const simple = await runtime.executor.run(task, { tier: "standalone" });
console.log(simple.executionTier, simple.finalText);

// 复杂 — SpAgent 内核
const complex = await runtime.executor.run(task, { tier: "kernel" });

// 自动 — 按 Profile 规则
const auto = await runtime.executor.run(task, { agentId: "reviewer" });
```

### 远程 HTTP Client

非 Node 宿主或跨进程调用：

```ts
import { OneAgentClient } from "@compo/oneagent";

const client = new OneAgentClient({
  baseUrl: "http://127.0.0.1:8790",
  authToken: process.env.ONEAGENT_TOKEN,
});

const { result } = await client.run({
  agentId: "reviewer",
  goal: "审阅合同第三节",
  tier: "standalone",
  personaOverrides: { "style.tone": "严谨" },
  contextRefs: [{ kind: "document", ref: "contract:v3" }],
});

console.log(result.finalText);
```

Python / Java 等语言：按相同 HTTP 契约自行封装，参考上方 curl 示例。

## Sidecar 部署

OneAgent 负责 Persona/Skills 层，**kernel tier** 转发至独立 SpAgent Gateway：

```yaml
# oneagent.config.yaml
kernel:
  mode: sidecar
  spagentUrl: http://127.0.0.1:8787
```

```bash
# 终端 1：SpAgent
cd ../SpAgent && OPENAI_API_KEY=... npm run example:http-gateway

# 终端 2：OneAgent
cd OneAgent && ./scripts/start.sh
```

standalone tier 仍在 OneAgent 进程内完成，不经过 SpAgent Gateway。

## 与 SpAgent 直接调用对比

| 需求 | 推荐 |
|------|------|
| 角色 / Persona / Skills | OneAgent `/v1/agents/:id/run` |
| 简单问答 / 审阅建议 | OneAgent `tier=standalone` |
| 多步工具 / 联邦 / 审批 | OneAgent `tier=kernel` 或直接 SpAgent `/v1/tasks/run` |
| 纯内核、无角色 | SpAgent `/v1/tasks/run` |

OneAgent 在任务进入 SpAgent 前完成 Profile enrichment（persona、capabilities、constraints、tier 路由）。

## 联邦外挂

配置 `federation.config` 或环境变量 `COMPO_PLUGINS_CONFIG` 指向 compo 插件 JSON，kernel tier 自动加载：

- MemoAgent（记忆）
- SkillForge（远程 Skills / 工具）
- knowledge-platform

standalone tier 不使用联邦工具，但 Persona/Skills 仍由 OneAgent 注入。

## Sidecar 流式

`kernel.mode: sidecar` 时，`POST /v1/agents/:id/stream` 会将 SSE 转发至 SpAgent `/v1/tasks/stream`：

```bash
curl -N -X POST http://127.0.0.1:8790/v1/agents/planner/stream \
  -H 'Content-Type: application/json' \
  -d '{"tier":"kernel","task":{"goal":"规划任务","actor":{"userId":"u1"}}}'
```

鉴权：设置 `CORE_AGENT_GATEWAY_TOKEN` 或 `ONEAGENT_SIDECAR_TOKEN`。

## Subagent Delegation

**kernel tier 内**（模型工具调用）：

```yaml
# agents/copilot.yaml
spec:
  delegation:
    allow: [planner, reviewer]
  capabilities:
    allow: [delegate_agent, ...]
```

**MCP 直接委派**（不经模型）：

```json
{ "fromAgentId": "copilot", "toAgentId": "planner", "goal": "拆解步骤" }
```

## 常见问题

**Q：standalone 能否调用工具？**  
不能。需工具时显式传 `tier: "kernel"` 或使用 `auto` + Profile escalate 规则。

**Q：stream 支持 standalone 吗？**  
不支持。流式请用 `tier: "kernel"` + `POST /v1/agents/:id/stream`。

**Q：如何强制复杂任务走 SpAgent？**  
传 `tier: "kernel"` 或 `metadata: { forceKernel: true }`。

**Q：HostBridge 在 standalone 下生效吗？**  
`resolveContext` 生效（contextRefs 注入 system）；`requestApproval` 仅 kernel tier 工具调用时需要。

**Q：Sidecar 模式支持流式吗？**  
支持。kernel tier 的 `/stream` 会转发 SpAgent Gateway SSE。

**Q：Subagent 委派需要什么 tier？**  
模型内 `delegate_agent` 工具需 kernel tier；MCP `oneagent_delegate` 无此限制。
