# Cocreator × Muiltchat 合并架构设计文档

> 合并日期：2026-05-06
> 版本：v1.0

---

## 一、项目概述

### 1.1 合并目标

将 **muiltchat** 的多模型对话/对比能力整合进 **Cocreator** 作为统一的「AI对话」入口，同时保留 Cocreator 完整的四大功能模块（来源管理 → 信息挖掘 → Skill共创 → 内容输出）。

### 1.2 技术栈

| 层级 | 原 Cocreator | 原 Muiltchat | 合并后 |
|------|-------------|-------------|--------|
| 前端 | Next.js (App Router) + TypeScript + Tailwind + shadcn/ui | React (Vite) + TypeScript | **Next.js (App Router)** + TypeScript + Tailwind + shadcn/ui |
| 后端 | Node.js + TypeScript + Prisma + SQLite | Python FastAPI + 多LLM客户端 | **Node.js + TypeScript** + Prisma + SQLite + 内置多LLM适配器 |
| LLM调用 | Cocreator llm-gateway (OpenAI/Claude/DeepSeek适配器) | Python 多LLM客户端工厂 (10+ providers) | **统一 LLM Gateway**，复用 Cocreator 架构 + 扩展 Muiltchat 的 providers |

### 1.3 核心设计原则

1. **Cocreator 作为主项目框架**：所有功能作为 Next.js App Router 的一个 Tab 页面或 API 路由
2. **muiltchat 多模型对话能力作为 Cocreator 的一个功能模块**：新增「AI对话」Tab
3. **不运行两个后端服务**：Python FastAPI 不独立运行，其核心逻辑（多LLM客户端）移植到 Node.js
4. **多Agent讨论模式**：作为 AI对话 Tab 的一个子模式

### 2.2 AI对话 Tab 页面结构

```
/projects/[id]/ai-chat/page.tsx
├── features/ai-chat/
│   ├── MultiModelChat.tsx      ← 主聊天组件（整合 muiltchat ChatArea）
│   ├── ChatSidebar.tsx          ← 对话列表侧边栏（整合 muiltchat Sidebar）
│   ├── ChatInput.tsx            ← 输入框
│   ├── MessageBubble.tsx        ← 消息气泡
│   ├── CompareView.tsx          ← Compare Mode 并排视图
│   ├── AgentDiscussion.tsx      ← 多Agent讨论视图
│   └── lib/
│       ├── chat-service.ts      ← API 调用（SSE流式）
│       ├── chat-store.ts         ← Zustand 状态管理
│       └── models.ts            ← 可用模型列表
```

### 2.3 与 Cocreator 的集成点

| 集成点 | 说明 |
|--------|------|
| **API 客户端** | 复用 `@/lib/api/client`，新增 `/api/v1/chat/stream` 路由 |
| **认证** | 复用 Cocreator 的 auth middleware，聊天会话关联 projectId |
| **Session 持久化** | 聊天记录存 Prisma `ConversationSession` 表，type='multi-chat' |
| **国际化** | 复用 Cocreator i18n（zh-CN / en-US） |

### 2.4 Compare Mode vs Single Mode vs Agent Discussion

**Single Mode**：一个模型回答，经典聊天界面
**Compare Mode**：多个模型并行回答，左右并排对比（muiltchat 的核心能力）
**Agent Discussion**：用户选择多个 Agent 角色，多轮对话讨论（手动调用）

### 3.3 多模型并行流式

`POST /api/v1/chat/stream` 支持同时向多个模型发送请求，每个模型的流独立返回，通过 SSE 的 `model_id` 字段区分。

请求体：
```json
{
  "model_ids": ["gpt-4o", "claude-4.6-sonnet"],
  "messages": [{"role": "user", "content": "..."}],
  "project_id": "uuid"
}
```

SSE 响应：
```
event: model_start
data: {"model_id": "gpt-4o"}

event: chunk
data: {"model_id": "gpt-4o", "content": "Hello"}

event: chunk
data: {"model_id": "claude-4.6-sonnet", "content": "Hi there"}

event: done
data: {"model_id": "gpt-4o", "usage": {...}}

event: done
data: {"model_id": "claude-4.6-sonnet", "usage": {...}}
```

### 3.4 Prisma Schema 扩展

在现有 `ConversationSession` 表增加 type 字段：
```prisma
model ConversationSession {
  id        String   @id @default(uuid())
  projectId String
  userId    String
  type      String   @default("skill")   // "skill" | "single" | "compare" | "agent-discuss"
  modelIds  String?                      // JSON数组 ["gpt-4o", "claude-4.6-sonnet"]
  ...
}
```

## 四、多Agent讨论模式设计

### 4.1 概念

**手动 Agent 调用**：用户手动选择多个 Agent 角色（如：技术专家、产品经理、竞品分析师），为每个 Agent 配置角色 Prompt，所有 Agent 依次收到用户问题并作答。

### 4.2 内置 Agent 角色

| Agent ID | 名称 | 角色描述 |
|----------|------|---------|
| `tech-expert` | 技术专家 | 从技术角度分析问题，关注架构、性能、实现细节 |
| `product-manager` | 产品经理 | 从市场和用户角度分析，关注需求、体验、商业价值 |
| `competitor-analyst` | 竞品分析师 | 分析竞品动态、市场格局、技术趋势 |
| `skeptic` | 质疑者 | 蓝军视角，质疑假设、识别风险、找出漏洞 |
| `synthesizer` | 综合分析师 | 整合各方观点，给出最终结论和行动建议 |

### 4.3 Agent Discussion 工作流程

1. 用户进入「AI对话」Tab，选择「Agent 讨论」模式
2. 用户选择 2-5 个 Agent 角色，输入问题
3. 系统依次调用每个 Agent（可并行），每个 Agent 以其角色身份作答
4. 所有 Agent 回答展示在同一视图（按角色分组）
5. 可选：用户追问 → 各 Agent 继续讨论

## 五、数据流设计

### 5.1 多模型聊天数据流

```
前端 ChatInput
  ↓ 发送消息
POST /api/v1/chat/stream
  ↓ 解析 model_ids
并行调用各 LLM 适配器 (stream_chat)
  ↓ 每个适配器独立 SSE 流
SSE Multiplexer（合并多流）
  ↓
前端 EventSource / fetch +ReadableStream
  ↓ 更新 Zustand store → 渲染 UI
```

### 5.2 Session 持久化数据流

```
前端 sendMessage
  ↓ 保存到 Zustand store（本地）
  ↓ 同时调用 API 保存到 Prisma
POST /api/v1/chat/sessions/:id/messages
  ↓
Prisma ConversationSession / ConversationMessage
```
